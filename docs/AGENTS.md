# Documentation

Документация для запуска и демонстрации стенда.

## Требования

- Документация должна быть практичной: команды запуска, endpoints, credentials, smoke checks, demo flow.
- Если меняются порты, имена сервисов, credentials, DAG id, topic names или vitrine names, обнови `README.md` и `docs/demo-runbook.md`.
- Явно помечай локальные demo credentials как небезопасные для production.
- Не добавляй длинные архитектурные эссе вместо шагов, полезных для демонстратора.
