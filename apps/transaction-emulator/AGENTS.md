# Transaction Emulator

Небольшой сервис фоновой генерации демо-нагрузки.

## Назначение

- Периодически создает транзакции через wallet API.
- Периодически отправляет clickstream-события через clickstream API.
- Дает поток данных для CDC, Kafka и витрин без ручного UI-сценария.

## Требования

- Не пиши напрямую в PostgreSQL или Kafka; эмулятор должен использовать публичные HTTP API.
- Частота нагрузки управляется переменной `TRANSACTIONS_PER_MINUTE` (по умолчанию 6, проброшена через `.env`).
- Пул пользователей задаётся `DEMO_USER_IDS` (CSV). Пользователь `00000000-0000-0000-0000-000000000001` зарезервирован для ручной демонстрации в UI и намеренно исключён из дефолтного пула, чтобы его баланс не «съезжал» от фонового эмулятора между демо-прогонами.
- Полностью отключается переменной `EMULATOR_ENABLED=false` — контейнер стартует и сразу завершается; используется для воспроизводимых демо без фонового трафика.
- По умолчанию нагрузка должна оставаться небольшой и подходящей для локального стенда на 20 GB RAM.
- Ошибки итерации логируй и продолжай цикл, чтобы временная недоступность сервисов не останавливала контейнер.
- Не добавляй сложный сценарный движок без явной задачи.

## Проверки

- `python3 -m py_compile apps/transaction-emulator/emulator.py`
- `docker compose -f infra/docker-compose.yml --env-file .env.example build transaction-emulator`
